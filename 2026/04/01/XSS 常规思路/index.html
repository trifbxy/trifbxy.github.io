一、XSS 思路介绍

XSS 的本质是攻击者将 恶意脚本代码 通过 Payload 注入到浏览器代码中，而开发者没有对用户输入进行充分的过滤和转义，导致原先的浏览器代码 结构和语义 发生 改变。

根本原因：

  • 开发者过于信任用户输入，没有进行充分 验证和转义。
  • 通过字符串拼接构造脚本代码，而非 模板引擎。
  • 将请求参数直接拼接到 危险执行函数 中，而非 DOM 操作 API。
  • 缺乏安全有效的 内容安全策略（CSP），难以限制恶意脚本执行。

防御思路：严格过滤输入、避免使用危险执行函数、使用安全模板引擎或 DOM 操作 API、配置有效的内容安全策略（CSP）。

XSS 思路：

  1. 寻找浏览器代码执行环境下的 注入点。
  2. 根据页面响应差异反推并绕过 防御策略。
  3. 尝试 Payload 探测 浏览器代码底层结构。
  4. 拼接、构造恶意脚本代码完成非法操作。
  5. 最终窃取用户敏感信息、劫持会话，甚至实现 持久化控制。

二、xss-labs靶场简介

xss-labs 是一个专门为学习 XSS（跨站脚本攻击） 而设计的开源靶场环境。它通过一系列由浅入深的关卡，帮助安全学习者和开发者理解 XSS 漏洞的触发原理、绕过技巧以及防御方法。

三、XSS 类型

根据恶意脚本的注入与触发方式，XSS 可分为以下三种类型：

类型 触发特点 典型场景
反射型 XSS 恶意脚本代码附加在 URL 参数中，需要诱导用户点击链接 搜索框、错误页面、URL 参数反射
存储型 XSS 恶意脚本代码被持久化存储在服务器数据库或文件中 评论区、留言板、个人资料、帖子内容
DOM 型 XSS 完全由前端 JavaScript 操作 DOM 时注入，不经过后端 innerHTML、eval、location.hash 操作

1. 反射型 XSS

触发方式：攻击者诱导受害者点击包含恶意脚本代码的 URL，服务器将恶意脚本代码拼接到浏览器代码中，导致受害者浏览器执行恶意脚本代码。

特点：

  • 恶意脚本代码 无法持久化，仅影响点击链接的用户。
  • 通常结合钓鱼网站、短链接、社工进行传播。
  • 探测难度低，响应包中可直接观察到注入点位置。

典型 Payload：

2. 存储型 XSS

触发方式：攻击者提交恶意脚本代码到服务器，使其存储到数据库。当其他受害者访问包含恶意脚本代码的页面时，恶意脚本代码从数据库中取出并执行。

特点：

  • 恶意脚本 持久化，影响所有访问该页面的用户。
  • 危害极大，可形成 XSS 蠕虫。
  • 常见于 UGC 功能。

典型 Payload：

3. DOM 型 XSS

触发方式：攻击者构造包含恶意脚本代码的 URL，当前端 JavaScript 读取 location.hash 并直接通过 innerHTML 插入到 DOM 时，恶意脚本代码执行。

特点：

  • 不经过后端，服务器响应包中无法发现异常。
  • 完全由前端代码漏洞导致。
  • 需要分析前端 JavaScript 源码才能定位注入点。

典型 Payload：

核心思路：

通过分析浏览器代码底层结构快速判断 XSS 类型，选择对应的探测与利用方式。反射型关注响应包分析，存储型关注浏览器代码底层结构猜解，DOM 型关注浏览器代码审计。
