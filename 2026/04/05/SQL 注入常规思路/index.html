SQL 注入常规思路

SQL 注入介绍

SQL 注入 的本质是攻击者构造 Payload 并注入到 SQL 中，导致 SQL 结构或语义 发生 改变，最终执行非预期的数据库操作。

根本原因：

  • 对请求参数缺乏充分的 过滤和转义。
  • 业务逻辑与安全校验 脱节，导致过滤机制失效。

SQL 注入思路介绍

SQL 注入思路：

  • 寻找可能与数据库交互的 注入点。
  • 根据页面响应差异反推并绕过 过滤机制。
  • 尝试 Payload 探测 SQL 底层结构。
  • 构造 Payload 获取 核心数据。

SQL 底层结构分析思路介绍（简单测试环境）

构造 Payload 为 False or True 构型，尝试改变 True 的 SQL 语义 判断注入点类型，尝试改变 False 的 SQL 结构 判断闭合符，观察页面表现差异。

根据传值方式分为 GET / POST，根据参数类型分为 数字型/字符型。
