发表于2026-03-21|安全技术
一、SQL 注入思路介绍SQL 注入的本质是攻击者将恶意 SQL 代码通过 Payload 插入到 SQL 语句中，而开发者没有对用户输入进行充分的过滤和转义，导致原先的 SQL 语句结构和语义发生改变。 根本原因： 开发者过于信任用户输入，没有进行充分验证和转义。 使用字符串拼接构造 SQL 语句，而非参数化查询。 数据库错误信息被直接返回给前端，为信息收集提供便利。 防御思路：严格过滤输入、使用参数化查询、不向用户暴露错误信息。 SQL注入思路： 寻找可能与数据库产生交互的注入点。 尝试 Payload 探测注入点类型和闭合符。 利用数据库错误信息或页面回显进行初步信息收集。 拼接、构造恶意 SQL 语句完成非法操作。 最终获取核心数据，甚至尝试获取服务器权限。 二、sqli-labs 靶场简介sqli-labs 是一个专门用于学习和练习 SQL 注入的开源靶场，每个关卡都专注于不同的注入场景和技巧，非常适合用来熟悉手工注入的各种姿势。 三、快速判断注入类型与闭合符（简单测试环境）在简单的测试环境中，根据传值方式和 Payload 的响应情况，通常可以通过以下四种场景...
